# /seed — 生成并验证种子数据

检查并生成满足课程验收硬指标的种子数据。

## 执行步骤

1. **检查当前数据量**
   读取 `src/backend/src/data/` 下所有数据文件，统计：
   - `spots.js` 中景点/校园数量（目标 ≥200）
   - 用户数量（目标 ≥10）
   - 道路图节点与边数量（目标边 ≥200）
   - 建筑物数量（目标 ≥20）
   - 设施类型（目标 ≥10）、设施实例（目标 ≥50）

2. **输出验收报告**
   以表格形式输出各项指标的当前值 vs 目标值，标记 ✓/✗。

3. **补充不足项**（如有缺口）
   - 不足的景点：程序化生成（覆盖不同城市、类型）
   - 不足的道路边：在已有景区图上随机补充节点和边
   - 保持数据格式与现有文件一致

4. **验证道路图连通性**
   对每个景区的道路图运行 BFS，确认至少随机选取 10 对节点可计算最短路。

运行命令：`node src/backend/src/data/validate.js`（如不存在则先创建）
