---
name: daily-reporter
description: 结合 GitHub MCP (远程) 和本地 Git (本地) 状态，生成全方位的每周开发日报。
---

请执行以下步骤来生成今日开发日报：

1. **获取变更记录 (双重检查模式)**：
   - **第一步 (检查远程 GitHub)**：
     - 使用 `github MCP` 工具调用 `list_commits`。
     - 筛选出最近一周的提交记录。
     - _注意：如果用户没有推送代码，这一步可能为空，请忽略错误继续下一步。_
   - **第二步 (检查本地工作区)**：
     - 运行 Shell 命令 `git status` 查看未提交的更改。
     - 运行 Shell 命令 `git diff HEAD` 查看本地已提交但未推送的更改（如果存在）。
     - 运行 Shell 命令 `git log --since="24 hours ago" --oneline` 作为本地提交的补充检查。

2. **读取关键文件内容**：
   - 根据上述步骤收集到的变更文件列表，读取核心代码文件（如 `.py`, `.js`, `.ts`, `.md`）的内容，以理解具体的业务逻辑。

3. **生成日报**：
   - 请在对话框中直接输出一份 Markdown 格式的日报，包含以下板块：
     - **📅 本周概览**：综合本地和远程信息，一句话总结今天的工作重心。
     - **✅ 已完成任务**：
       - 列出 3-5 个核心产出。
       - _如果代码在本地未提交，标记为 "(本地开发中)"_。
       - _如果代码已推送到 GitHub，标记为 "(已推送)"_。
     - **🐛 问题与解决**：分析代码变更，识别修复的 Bug 或技术难点。
     - **📂 变更文件清单**：列出涉及的主要文件路径。
     - **🚀 明日计划**：根据当前进度，推测或建议明天的开发方向。

4. **容错处理**：
   - 如果 GitHub MCP 调用失败（例如未联网或无权限），仅依赖本地 Git 命令生成报告，并在开头提示“仅基于本地记录生成”。
