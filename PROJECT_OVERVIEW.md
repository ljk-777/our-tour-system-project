# 旅游系统 — 项目概览

## 一、项目需求

以**旅游地点推荐、路线规划、场所查询、旅游日记**为核心，扩展支持美食推荐、群组共享、社交动态、AI 行程微调、回忆录动画等功能。

### 核心功能

| 功能          | 说明                                                    |
| ------------- | ------------------------------------------------------- |
| 旅游推荐      | 按热度/评价/兴趣推荐景点/校园，Top10                    |
| 路线规划      | 单点最短路、多点往返、交通工具混合、室内导航            |
| 场所查询      | 附近设施按**道路距离**排序（禁止直线距离）              |
| 景点/建筑展示 | 名称、简介、标签、开放信息                              |
| 日记管理      | 发布、热度、评分、排序推荐                              |
| 日记交流      | 目的地筛选、标题精确查找、全文检索、压缩存储、AIGC 动画 |

### 扩展功能（根据调查问卷决定）

美食推荐 / 多人群组共享 / 同行人冲突协调 / 旅行动态广场 / 实时天气交通推送 / AI 实时行程微调 / 行李清单 / 红黑榜 / 回忆录动画

### 验收指标

- 景区/校园 ≥ 200 个
- 典型演示景区内部建筑 ≥ 20 个
- 设施类型 ≥ 10 种，实例 ≥ 50 个
- 道路图边数 ≥ 200 条
- 系统用户 ≥ 10 人

## 二、简要设计方案

### 架构

```
前端 SPA (React + Vite)
      ↕ REST API / WebSocket
后端 API (Node.js + Express)
      ↕
数据层 (PostgreSQL + Redis)
```

### 核心算法（每个功能独立实现并对比）

| 功能      | 算法方案                                 |
| --------- | ---------------------------------------- |
| 推荐/排序 | 小顶堆 TopK vs Quickselect vs 全量排序   |
| 最短路    | Dijkstra vs A\*（距离/时间双策略）       |
| 多点路线  | 最近邻 + 2-opt vs Held-Karp DP           |
| 场所查询  | 网格/四叉树候选 + 单源 Dijkstra          |
| 全文检索  | KMP/BM vs 倒排索引（+可选 Aho-Corasick） |
| 压缩存储  | Huffman vs LZ77/LZ78                     |
| 模糊查找  | BK-Tree vs Trie + 编辑距离               |
| 冲突协调  | 加权满意度贪心 vs 约束优先级过滤         |

### 前端关键页面

- 三维地球首页（目的地发现 + 路线预览）
- 地图导航页（路径高亮 + 策略切换 + 室内楼层）
- 日记社区（瀑布流 + 全文检索高亮）
- 动态广场（旅行动态流 + 点赞评论）
- 我的旅行（行程时间线 + 回忆录播放器）

## 三、开发环境

| 类别     | 技术栈                                           |
| -------- | ------------------------------------------------ |
| 前端     | React 18 + Vite 5 + Tailwind CSS + React Leaflet |
| 后端     | Node.js + Express 4                              |
| 数据库   | PostgreSQL（持久化）+ Redis（缓存/推送）         |
| 地图     | Leaflet（2D 路线图）                             |
| 实时通信 | WebSocket（群组协作/实时推送）                   |
| 运行端口 | 前端 :5173，后端 :3001                           |

**启动方式**

```bash
# 后端
cd backend && npm run dev

# 前端
cd frontend && npm run dev
```
