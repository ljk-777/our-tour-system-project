# 第十四周项目周报

## 一、本周项目整体进展

本周（6月7日—6月13日）以功能收尾和质量加固为主线。积压七周的 KMP 检索前端可见化终于在本周落地，李佳坤实现了日记与景点搜索的关键词高亮渲染，算法价值正式从后端浮现至页面；戴鸿完成 Globe 多角度稳定性终验并修复极端视角下城市标注消失的边界 bug；万子豪主导系统全链路回归测试，记录并修复多处前端边界异常；缑文轩完成 Trie 前缀联想前端联动，并持续推进 PR 合并管理与代码审查。至本周末，四条核心算法均已在页面层面可见。

## 二、本周各模块进展

### 1. KMP 检索前端可见化（`src/frontend/src/pages/Diary.jsx`、`src/frontend/src/components/SearchHighlight.jsx`）
- 新增 `SearchHighlight` 组件：接收关键词与原文，调用后端 KMP 接口后将匹配片段高亮渲染（李佳坤）
- 日记列表搜索框接入 KMP 高亮，匹配字符以橙色背景标注，支持多关键词
- 景点搜索页同步集成，搜索结果卡片标题与描述均显示高亮片段
- 后端 `/api/search/kmp` 接口补充测试用例，覆盖中文分词、空串、重叠匹配边界场景

### 2. Trie 前缀联想前端联动（`src/frontend/src/pages/Spots.jsx`、`src/backend/src/algorithms/trie.js`）
- 景点搜索框接入 Trie 前缀自动补全（缑文轩）：输入 2 字符后触发联想下拉
- 后端暴露 `/api/spots/suggest` 接口，复用已有 Trie 实现，返回前缀匹配结果
- 优化 Trie 节点插入逻辑，支持景点名称与城市名双重索引

### 3. Globe 稳定性终验（`src/frontend/src/components/Earth3D.jsx`、`GlobeOverlay.jsx`）
- 多角度（赤道/极区/高纬度）城市点击旋转全量测试（戴鸿），确认无旋转越界与镜头跳变
- 修复城市标注在极小 FOV 下偶发消失的边界问题
- 流星弹幕低配设备压力测试（4核/8GB），帧率稳定维持 55fps 以上

### 4. 系统全链路回归测试（`tests/`）
- 走通注册→登录→景点浏览→收藏→群组创建→共享景点→路线规划完整用户路径（万子豪）
- 修复 Profile 页足迹地图在 Safari 浏览器的渲染兼容性问题
- 确认群组 Dijkstra 路线、目的地投票、退出群组在主流浏览器下全部正常
- 修复美食推荐页 TopK 结果排序在极少数情况下不稳定的边界问题

### 5. 代码质量与文档收尾

## 四、当前存在的问题

1. **KMP 高亮覆盖范围有限** — 目前仅覆盖日记与景点搜索，路线搜索、美食搜索尚未集成高亮，属于遗留技术债。

2. **adversarial/ 与 mcp/ 始终未入主分支（连续八周）** — 对抗测试脚本和 MCP 服务器从未真实运行过，review-agent 仍只是空目录，课程设计结束后如有机会应补入。

3. **Globe 弹幕低配帧率未全量验证** — 4核/4GB 以下设备未测试，极低配环境下的帧率表现尚不明确。

4. **Trie 联想仅支持景点名称** — 城市名与标签的前缀检索尚未集成，联想覆盖范围偏窄。
