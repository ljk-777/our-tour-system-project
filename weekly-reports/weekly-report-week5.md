# 第五周项目周报

## 一、本周项目整体进展

本周项目进入了“前后端整合与核心功能铺设”阶段，整体完成了较大规模的架构合并与功能扩展。小组将前期已有代码与外部整合资源进行了统一重构，进一步明确了前后端分层结构，并补充了多项课程设计所需的核心算法与业务页面。

目前，项目已经基本形成了“前端页面 + 后端接口 + 算法模块 + 数据层”的整体雏形，能够支撑后续继续完善旅游推荐、路线规划、场所查询、旅游日记交流等主线功能。同时，本周还对若干影响演示与后续开发的 Bug 进行了集中修复，为下一阶段的功能打通和页面联调奠定了基础。

---

## 二、本周各模块进展

### 1. 项目架构与代码整合
- 完成外部项目压缩包内容的解压与整合，将前后端代码分别并入 backend/ 与 frontend/ 目录。
- 初步形成了较清晰的前后端分层结构，后端开始采用 Repository 数据访问层思路组织代码。
- RESTful API 路由结构进一步完善，为景点、路线、日记、用户等模块提供统一接口基础。

### 2. 核心算法模块补充
本周补充并接入了多项与课程设计主线密切相关的算法实现，包括：
- Dijkstra 最短路径算法
- Heap 最小堆实现
- KMP 字符串匹配算法
- Trie 前缀树（用于搜索联想）

这些算法模块的加入，使项目不仅具备页面展示能力，也开始具备课程验收所强调的“自主数据结构与算法实现”基础。

### 3. 功能页面与业务能力扩展
- 新增管理后台页面（/admin）
- 新增美食推荐页面（/foods）
- 完善日记评论与删除相关功能
- 增强路线规划页面的路径可视化效果
- 补充项目概述文档（PROJECT_OVERVIEW.md）

### 4. Bug 修复与技术改进
本周还集中处理了若干影响功能联调和用户体验的问题，包括：
- 导航栏切换后的滚动位置问题
- AuthContext 登录/注册返回格式问题
- 日记评论字段类型转换问题
- 路线图距离数据修正（如灵隐寺→香山）
- 暗色主题下输入框可见性问题
- 点赞重复点击限制
- “展开全文”按钮阈值优化

同时，技术层面还进行了若干改进：
- 后端补充了全局错误处理中间件、请求日志与输入验证
- 前端优化了权限系统的视觉区分与 API 封装方式

3. 当前部分新增页面仍偏展示型，后续需要继续与课程核心算法和业务逻辑深度结合。
4. 项目代码量增长较快，后续需要进一步注意模块边界与可维护性。
